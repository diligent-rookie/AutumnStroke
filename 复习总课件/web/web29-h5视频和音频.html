视频和音频

  1. 标签audio 、video
    原来的flash手机不兼容，而且需要插件才能运行
    audio 、video（用src链接地址，但是不可控制，需要媒体元素，但是控件的样式浏览器里也不同）
    source（为了做兼容，不将src放在媒体标签上，而是放在这个标签上，这样可以自动匹配支持的格式，不支持自动跳过）
  2. 视频容器：容器文件，类似于压缩了一组文件；音频轨道；视频轨道；元数据：封面，标题，字幕等；格式：.avi、.flv、.mp4、.mkv、.ogv等
  3. 使用方法：
        <audio controls src="johann_sebastian_bach_air.mp3"></audio>
    <video src="Intermission-Walk-in_512kb.mp4"></video>
    
    <video controls>
        <source src="Intermission-Walk-in.ogv"></source>
        <source src="Intermission-Walk-in_512kb.mp4"></source>
    </video>
  4. 编解码器：原始的视频容器非常大，添加需编码（减少加载时间，质量小），播放需解码（重新变成可播放视频音频），这些编码解码器有收费的有免费的，每个浏览器用的也不同，所以浏览器支持的格式也不同
    常见音频编解码器：AAC、MPEG-3、Ogg Vorbis
    常见视频编解码器：H.264、VP8、Ogg Theora

媒体元素属性、方法和事件

  1. 属性：
    controls : 显示或隐藏用户控制界面
    autoplay : 媒体是否自动播放
    loop : 媒体是否循环播放
    currentTime : 开始到播放现在所用的时间(用js获取或设置)
    duration : 媒体总时间(只读)
    volume : 0.0-1.0的音量相对值（0为静音）
    muted : 是否静音（静音为true）
    autobuffer : 开始的时候是否缓冲加载，autoplay的时候，忽略此属性
    paused : 媒体是否暂停(只读，暂停为true)
    ended : 媒体是否播放完毕(只读，结束为true)
    error : 媒体发生错误的时候，返回错误代码 (只读，没有错误则为null)
    currentSrc : 以字符串的形式返回媒体地址(只读)
  2. 方法：
    play() : 媒体播放
    pause() : 媒体暂停
    load() : 重新加载媒体（比如你修改了source的src视频地址，如果不使用load方法重新加载，仍然显示的之前的视频）
  3. 事件：
    progress //当浏览器正在获取媒介数据时运行的脚本
    suspend //在媒介数据完全加载之前不论何种原因终止取回媒介数据时运行的脚本（挂起）
    abort //在退出时运行的脚本
    error //当在文件加载期间发生错误时运行的脚本
    stalled //在浏览器不论何种原因未能取回媒介数据时运行的脚本
    play //当媒介已就绪可以开始播放时运行的脚本
    pause //当媒介被用户或程序暂停时运行的脚本
    loadedmetadata //当元数据（比如分辨率和时长）被加载时运行的脚本
    loadstart //在文件开始加载且未实际加载任何数据前运行的脚本
    loadeddata //当媒介数据已加载时运行的脚本
    waiting //当媒介已停止播放但打算继续播放时（比如当媒介暂停已缓冲更多数据）运行脚本
    playing //当媒介已开始播放时运行的脚本
    canplay //当文件就绪可以开始播放时运行的脚本（缓冲已足够开始时）
    canplaythrough //当媒介能够无需因缓冲而停止即可播放至结尾时运行的脚本
    seeking //seeking属性变为true，浏览器正在请求数据
    seeked //seeking属性变为false，浏览器停止请求数据
    timeupdate //当播放位置改变时（比如当用户快进到媒介中一个不同的位置时）运行的脚本
    ended //当媒介已到达结尾时运行的脚本（如发送'感谢观看'）
    ratechange //每当回放速率改变时运行的脚本（比如当用户切换到慢动作或快进模式）
    readystatechange //每当就绪状态改变时运行的脚本（就绪状态监测媒介数据的状态）
    durationchange //当媒介长度改变时运行的脚本
    volumechange //每当音量改变时（包括将音量设置为静音）时运行的脚本
    emptied //当发生故障并且文件突然不可用时运行的脚本（比如连接意外断开时或用load重新加载了媒体）
  4. video额外属性：
    poster : 视频播放前的预览图片
    width、height : 设置视频的尺寸
    videoWidth、 videoHeight : 视频的实际尺寸(只读)
