变量和命名规则：变量就是用于存储信息的容器

  1. 没人天天叫毕加索的全名，js里也可以用一个名字储存数据，将事件加在名字上可以大大减少代码量
  2. 变量的命名规则：变量必须以字母开头；虽然变量也能以$和_符号开头（不过不推荐这么做）；变量名称也对大小写敏感（a和A是不同的变量)。
  3. 变量使用var关键字声明：var 名字 = ，后面可以是数字，字符串或者一个对象等等
  4. 好的代码编写习惯是总是用var声明变量的，并且始终存放相同类型的值，常用的命名规则：
    • Camel标记法：首字母是小写的，接下来的单词都以大写字母开头。
    • Pascal标记法：首字母是大写的，接下来的单词都以大写字母开头。
    • 匈牙利标记法：在以Pascal标记法命名的变量前附加一个小写字母说明该变量的类型。如 i表示整数,s表示字符串,b代表boolean值

数据类型：JavaScript中有5种原始数据类型——Undefined（未定义）、Null（空对象）、Boolean（布尔值）、Number（数字）、String（字符串）和特殊一个引用型类型——Object（对象）

  1. Undefined（未定义）这个值表示变量不含值
  2. Null（空对象）可以通过设置这个值清空变量
  3. Boolean（布尔值）只有true和false两个值
  4. Number（数字）不分整数和浮点
  5. String（字符串）用双引号或单引号括起来的部分

属性操作：

  1. <input id="btn1" type="button"/>
    上面元素的属性名（id），属性值（btn1）
    读：获取某元素属性值（元素.属性）
    写：替换某元素属性值（元素.属性="新值"）替换操作
  2. 获取的属性的值都是以字符串形式显示，“+”可以用来做字符串的拼接
  3. 而delete操作符是删除属性的唯一方法。把属性设置为undefined或者null只是移除了属性和值的关联
    语法：delete 元素.属性名
  4. 元素.innerHTML：元素的内容属性，用以读取页面中元素的所有html内容
    写入：元素.innerHTML="新内容"
    内容其实就是html文件，可以用以加入新的标签
  5. "[]"的一种用法类似"."
    "."后面的值不能变（如.style.width的width不能用变量）
    而"[]"里面的值可以变（如.style.width可以写成.style["width"]，或者用变量，通过修改变量改变要修改的样式如：.style[attr]
  6. 鼠标左键抬起：onmouseup
  7. 鼠标移动：onmousemove ....

自增/自减：num+=x; num++;（num+=1的简写）/num-=x; num--;（num-=1的简写）

js中的css写法：js里不能出现"-"，如font-size需写成fontSize

class和关键字、保留字：可以使用class来同时改变多个样式，但class应写成className
关键字：js里用到的单词（如：function var）
保留字：js里保留的单词（如：class ）
不要使用关键字和保留字来给变量命名

if判断：

  1. if(条件){} //满足条件时。。。
  2. if(条件){}else{} //满足条件时。。。否则。。。
  3. if(条件1){}else if(条件2){}......else{}//满足条件时。。。否则如果。。。否则如果。。。
  4. 判断的注意事项：
    • js获取元素的href/src不是相对路径而是绝对路径，因此不能用来做判断
    • 颜色值也不能用来做判断（因为有很多颜色值的输出方式，如英文、rgb和16进制数）
    • innerHTML也不能用来做判断（在低版本浏览器上会有兼容性的问题）
    • 当需要判断上面这些值时可以使用布尔值开关的方式解决问题
    • 当需要判断内容是否相等时，需要用"=="（数值相等）或"==="（绝对相等—类型数值都相等，推荐），一个等号"="叫做赋值操作，意思是将等号右侧的值赋给左侧

属性兼容性问题：

  1. 直接通过js去修改input的type值（ie6-ie8不兼容）
    解决方法：可以绕过该方法，利用其他方法去做，如隐藏原有input框，显示新的input
  2. 没有style.float的这种写法，style.styleFloat是ie写法；style.cssFloat是其他浏览器写法
    解决方法：也可以绕过该方法，例如用添加class的方式去修改样式
  3. 因此当你需要解决兼容问题时，要么知道做如何解决兼容问题（如封装带有兼容问题的代码），要么会使用其他的解决方案（如上面的添加class的方法解决绕过该问题）
